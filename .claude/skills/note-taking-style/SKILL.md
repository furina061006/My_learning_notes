# 笔记风格 Skill

## 目标

本 Skill 要求 AI 在创作笔记时完全模仿 furina061006 的个人风格。这是**强制风格规范**，不是建议。

**核心理念**：笔记不是冷冰冰的文档，而是有温度的学习旅程。

---

## 一、语言风格（必须遵守）

### 1.1 中文为主，英文术语自然嵌入
- 不要刻意翻译术语：说"CPU"不说"中央处理器"，说"GPIO"不说"通用输入输出口"
- 专有名词保持原样大写：STM32、PWM、DMA

### 1.2 口语化、亲切感
- 多用**"我们"**：*"下面我们开始讲述"*
- 用**设问句**自问自答：*"Q: 链接时发生了什么？"*
- 像给朋友讲解一样自然

### 1.3 幽默感
- **Emoji**：😵 表示头大/没时间，🙏 表示求轻喷，😋 表示开玩笑，🤔 表示不确定
- **删除线**：~~其实和上一点同源~~、~~因为看不懂~~
- **自嘲**："没时间写了😵"、"孩子累了，截图偷个懒()"
- **括号吐槽**：（但其实不能用中文）

### 1.4 坦诚不懂
- 不懂就标出来：*"彻底AI了，现在看不懂😋"*
- 不确定就标注：*"🤔这一块我不确定AI说的好不好"*
- 有错认错：*"坏啦, 前几天都讲错啦"*

### 1.5 强调标记
- 重要提醒用 `<span style="color:red">注:</span>`
- 用 `***` 或 `---` 做视觉分隔

---

## 二、结构组织

### 2.1 总-分结构
每篇笔记最前面放 TOC 目录，Markdown 列表层级呈现。

### 2.2 编号体系
```
一级用数字：1. 2. 3.
二级用数字：1.1 2.2
三级用中文：一、二、
```

### 2.3 Q&A 模式
关键概念用问答引出：
```markdown
### Q: 链接时发生了什么？
### A:
> 链接器为每个函数/全局变量分配最终的虚拟地址...
```

### 2.4 目录
TOC 用原生 Markdown 列表，标题变更时同步更新目录。

---

## 三、视觉元素

### 3.1 表格对比
善用表格做概念对比、参数罗列、分步说明。

### 3.2 ASCII 框图
流程/架构/数据结构用 ASCII art 画框图：
```text
┌──────────────────────┐
│       标题           │
├──────────────────────┤
│ 步骤1: ...           │
└──────────────────────┘
```

### 3.3 代码示例
C 代码用 ` ```c ` 标注，关键行加注释。

### 3.4 引用块
- 关键概念用 `>` 引用
- 警告用红色 HTML 标签

---

## 四、教学方法论

### 4.1 "加一层" 哲学
贯穿始终：每一层抽象都为了解决上一层的问题。
```markdown
众所周知 CPU 只能读 0/1，所以应该——**加一层！**
这个加一层的结果，就是汇编语言。
```

### 4.2 从具象到抽象
- 先给具体例子，再总结规律
- 先展示错误用法，再给出正确用法

### 4.3 嵌入式特化
- 时刻联系嵌入式场景（STM32 为例）
- 给出芯片具体参数（如 STM32F103C8T6：64KB Flash，20KB SRAM）

### 4.4 代码与结果对照
贴源码 → 贴输出/汇编，或提供跳转链接。

---

## 五、文件规范

- 中文文件名表意清晰：`C语言杂项笔记.md`
- 摘要/引言用 blockquote 开头，说明背景
- 大笔记开头加提示语（如视频教程链接）

---

## 六、禁止事项（违反即扣分）

- ❌ 不要写纯英文笔记
- ❌ 不要用正式学术口吻（那是 AI 的默认风格，不是用户的）
- ❌ 不要删除用户原有的 emoji 和吐槽
- ❌ 不要补充用户没学过的内容而不做标注
- ❌ 不要用 AI 的口头禅："作为AI"、"根据我的理解"、"值得注意的是"

## 适用场景

| 场景 | 说明 |
|------|------|
| 新建笔记 | 按本风格撰写全新学习笔记 |
| 扩写/完善 | 补充内容时保持风格一致 |
| 校对/润色 | 确保新内容不打破原有风格 |
