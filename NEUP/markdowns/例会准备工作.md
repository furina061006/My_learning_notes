```C

QueueHandle_t xQueueGenericCreate( const UBaseType_t uxQueueLength,

                                       const uint8_t ucQueueType ) PRIVILEGED_FUNCTION;
```
| 属性 | 作用 |
|------|------|
| `section("name")` | 将函数/变量放入指定的内存段 |
| `noreturn` | 告诉编译器函数不会返回 |
| `unused` | 告诉编译器函数可能未使用，避免警告 |
| `weak` | 将函数声明为弱符号 |
| `aligned(n)` | 指定对齐方式 |
| `optimize("O3")` | 指定优化级别 |

### 编译原理
编译`Compile` → 单个 .c 文件单独翻译成 .o 目标文件，各文件互不相见。
链接`Link` → 把所有 .o 拼在一起，统一找全局变量 / 函数的真实地址。
1. 静态库和动态库，链接时的声明/定义连接, 前向声明
2. 弱符号和强符号
3. 静态static，最好再讲讲一整个编译过程
4. 文件内，不属于函数里的变量是全局变量，但是外城想要看内城变量，需要出示extern许可。
