# 笔记风格 Skill

## 目标

本 Skill 记录了 furina061006 的个性化笔记风格。任何由 AI 协助新建或编辑的笔记文件都应遵循此风格，确保知识库的一致性和鲜明的个人特色。

**核心理念**：笔记不是冷冰冰的文档，而是**有温度的学习旅程**。

---

## 一、语言风格

### 1.1 中文为主，英文技术术语自然嵌入

```markdown
CPU 从 Flash 的 .text 段里读取指令，执行函数
```

- 不要刻意翻译术语（不要说"中央处理器"而说"CPU"）
- 英文术语保持原样，不需要斜体或引号
- 专有名词如 STM32、GPIO、PWM 等保持大写

### 1.2 口语化、亲切感

- 多用**"我们"**拉近距离：*"下面我们开始讲述"*
- 使用**设问句**自问自答：*"Q: 链接时发生了什么？"*
- 像给朋友讲解一样自然：*"话不多说，我们下面开始讲述："*

### 1.3 幽默感与 personality

适当穿插以下元素增加可读性：

| 元素 | 示例 | 说明 |
|------|------|------|
| Emoji | 😵🙏😋🤔😡👆 | 表达情绪，缓解枯燥 |
| 删除线 | ~~其实和上一点同源~~ | 吐槽、补充、幽默 |
| 自嘲 | "没时间写了😵"、"孩子累了，截图偷个懒()" | 坦诚地表达状态 |
| 网络用语 | "懂得都懂"、"光说不练假把式，上家伙()" | 接地气 |
| 括号吐槽 | （但其实不能用中文） | 括号内放轻松的补充 |

### 1.4 坦诚不懂

- 不懂就是不懂：*"彻底AI了，现在看不懂，到时候寒假了边学边理解😋"*
- 不确定就标注：*"🤔不知道大佬们有什么其他推荐"*
- 有错认错：*"坏啦, 前几天都讲错啦, 今天觉得太奇怪了又问了下才知道"*

### 1.5 强调与标记

- 重要提示用 `<span style="color:red">注:</span>`（红色标记）
- 有时用蓝色：`<span style="color:blue">(如下以 win端调用g++为例)</span>`
- 用 `***` 或 `---` 做视觉分隔

## 二、结构组织

### 2.1 总-分结构

每篇笔记最前面有 **目录（TOC）**，以 Markdown 原生列表呈现，层级保留：

```markdown
# 标题
## 目录
- [标题](#标题)
  - [目录](#目录)
  - [一级标题](#一级标题)
    - [二级标题](#二级标题)
```

### 2.2 编号体系

一级用数字序号：`1.`、`2.`、`3.` ...
二级用数字序号：`1.1`、`2.2` ...
三级用中文：`一、`、`二、` ...

### 2.3 Q&A 模式

关键概念用问答形式引出，模拟真实学习过程中的疑惑：

```markdown
### Q: 链接时发生了什么？
### A:
> 链接器为每个函数/全局变量分配最终的虚拟地址...
```

## 三、视觉元素

### 3.1 表格对比

善用表格做对比，尤其是：
- 概念对比（如静态库 vs 动态库）
- 参数罗列（如寄存器、类型说明）
- 分步说明

```markdown
| 维度 | 静态库 (.a) | 动态库 (.so) |
|------|------------|------------|
| 部署 | 独立性强 | 节省磁盘空间 |
```

### 3.2 ASCII 框图

对于流程、架构、数据结构等，用 ASCII art 画框图：

```text
┌──────────────────────────────────────────────────────┐
│                       说明标题                         │
├──────────────────────────────────────────────────────┤
│  步骤1: ...                                           │
│  步骤2: ...                                           │
└──────────────────────────────────────────────────────┘
```

### 3.3 代码示例

C 代码用 \`\`\`c 标注，并用注释解释关键行。代码块内可以嵌入 ASCII 示意图：

```c
// 关键行注释
int *p = (int *)malloc(sizeof(int));  // 动态分配 → 堆
```

### 3.4 引用块

- 关键概念、需要强调的原理用 `>` 引用块
- 警告用带颜色的 HTML 标签

```markdown
> 纯文本替换，只是在预处理阶段，把下文中的 `NAME` 文本全部换成 `value` 文本
```

## 四、教学方法论

### 4.1 "加一层" 哲学

贯穿始终的思维方式：每一层抽象都为了解决上一层的问题而存在。

```markdown
众所周知 CPU 只能读 0/1，所以应该——**加一层！**
这个加一层的结果，就是汇编语言。
```

### 4.2 从具象到抽象

- 先给具体例子，再总结规律
- 先展示错误用法，再给出正确用法
- 先跑通再优化

### 4.3 嵌入式中特化

- 时刻联系嵌入式场景（STM32 为例）
- 解释概念时指出在嵌入式中的特殊性
- 给出芯片具体参数（如 STM32F103C8T6：64KB Flash，20KB SRAM）

### 4.4 代码与结果对照

- 贴代码 → 贴输出/结果
- 贴源码 → 贴汇编/反汇编
- 跳转到示例文件：`跳转至[源码展示](../examples/example1/srcs/a.c)`

## 五、文档规范

### 5.1 文件命名

- 中文文件名，表意清晰：`C语言杂项笔记.md`、`寒假补完计划.md`
- 不需要英文命名或日期前缀

### 5.2 元信息习惯

- 标题用 `#` 一级标题
- 摘要/引言用 blockquote 开头，说明背景或动机
- 对于大笔记，开头加提示语（如江协科技视频链接）

### 5.3 目录维护

- TOC 保持与正文同步
- 标题变更时更新目录

## 六、禁止事项

- ❌ 不要写纯英文笔记（除非用户明确要求）
- ❌ 不要用过于正式的学术口吻
- ❌ 不要删除用户原有的 emoji 和吐槽
- ❌ 不要补充用户没学过的内容而不做标注
- ❌ 不要用 AI 的语气说"作为AI"、"根据我的理解"等
- ❌ 不要模仿标准教材的语调（那是 AI 的默认风格，不是用户的）
